## 2024-03-09 - React Native Loop Memory Leak & List Re-renders
**Learning:** React Native's `Animated.loop` can cause memory leaks and unintended background JS thread activity if not explicitly stopped when a component unmounts or a dependency (like an item's state that disables the animation) changes. Additionally, rendering custom touchable rows from an array using `map` without `React.memo` and stable callback props (like `useCallback`) forces the entire list to re-render when a single item's state changes.
**Action:** Always capture the reference returned by `Animated.loop(...)` and call `.stop()` in the `useEffect` cleanup function. When iterating over arrays to create UI rows, wrap the row component in `React.memo` and pass stable callbacks using `useCallback` to prevent O(n) rendering work.
